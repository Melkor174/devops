# Git для самых маленьких (будущих DevOps-инженеров)

> Git — это как «машина времени» для твоих файлов. Ты можешь сохранить состояние папки сегодня, потом что-то испортить, а затем вернуться к сохранённой версии.

## Зачем это нужно, если я просто учусь?

Представь, что ты ведёшь конспекты. Сегодня написал 3 страницы, завтра — ещё 2. Через неделю решил посмотреть, что было на первой странице. Без Git пришлось бы листать руками или хранить кучу копий типа `конспект_финал_реально_финал_v3.doc`.

С Git ты просто делаешь «сохранения» (коммиты) и в любой момент можешь посмотреть любое из них или вернуться назад.

## Как установить Git (один раз и забыть)

**Windows:**  
Зайди на сайт git-scm.com, скачай установщик, запусти его. Везде нажимай «Далее» — там всё уже настроено правильно.

**macOS:**  
Открой терминал (Programs → Утилиты → Терминал) и введи:  
`brew install git`  
Если пишет «brew not found» — скачай установщик с git-scm.com.

**Linux (Ubuntu/Debian):**  
Открой терминал и введи:  
`sudo apt install git`

**Проверка, что Git установился:**  
Открой терминал (или Git Bash на Windows) и введи:  
`git --version`  
Ты увидишь что-то вроде `git version 2.40.0` — значит всё ок.

## Первая настройка (представься Git)

Git хочет знать, кто делает сохранения. Введи в терминале по очереди две команды (только замени имя и почту на свои):

```bash
git config --global user.name "Иван Петров"
git config --global user.email "ivan@example.com"


1. git init — включить Git в папке
Что делает: Создаёт внутри текущей папки скрытую папку .git. С этого момента Git начинает следить за всем, что происходит в этой папке.

Как использовать:
Зайди в папку с конспектами и напиши:

git init

2. git status — посмотреть, что изменилось
Что делает: Показывает список файлов, которые ты изменил, добавил или удалил, но Git ещё не запомнил.

Как использовать:
git status


3. git add — сказать Git «вот эти изменения надо запомнить»
Что делает: Берёт файл (или папку) и кладёт его в «корзину для следующего сохранения». Git пока ничего не сохранил, просто взял файл на карандаш.

Как использовать:
git add имя_файла.txt     # добавить один файл
git add .                 # добавить ВСЕ файлы в папке

4. git commit — наконец-то сохранить версию
Что делает: Создаёт точную копию (снапшот) всех файлов, которые ты добавил через git add. Эта копия сохраняется навсегда.

Как использовать:

git commit -m "Кратко объясни, что изменилось"
Простыми словами: «Git, запомни всё, что я накидал в корзину (через add), и сделай это новой версией. И вот тебе подпись, что тут было сделано».

Почему -m: Буква m означает message (сообщение). Без него Git откроет какой-то непонятный редактор — новичкам это не нужно.

5. git log — посмотреть историю сохранений
Что делает: Показывает список всех коммитов (сохранений), которые ты сделал: кто, когда, с каким сообщением.

Как использовать:

bash
git log
Простыми словами: «Git, покажи всю историю моих сохранений — что я делал, когда и с какими комментариями».


git init

# 3. Создай первый файл
echo "Привет, Git!" > hello.txt

# 4. Посмотри — Git увидел новый файл?
git status
# Ты увидишь что-то вроде "untracked file: hello.txt"

# 5. Скажи Git, что этот файл надо запомнить
git add hello.txt

# 6. Сохрани первую версию
git commit -m "Первый коммит: добавил hello.txt"

# 7. Посмотри историю
git log
# Ты увидишь свой коммит с именем, почтой и сообщением

# 8. Теперь измени файл
echo "Вторая строчка" >> hello.txt

# 9. Посмотри — Git заметил изменение?
git status
# Теперь hello.txt красным — он изменён, но не добавлен

# 10. Добавь изменение и сохрани новую версию
git add hello.txt
git commit -m "Добавил вторую строчку"

# 11. Снова посмотри историю — теперь там два коммита
git log



Часто задаваемые вопросы (как есть)
Вопрос: Я случайно сделал коммит с неправильным сообщением. Что делать?
Ответ: Ничего страшного. В следующий раз напиши нормально. Сообщение можно исправить, но для первого месяца тебе это не понадобится.

Вопрос: А что будет, если я сделаю git add ., добавлю всё, а потом передумаю?
Ответ: Git пока ничего не сохранил, только приготовил. Можно очистить корзину командой git reset, но пока просто делай коммит — лучше лишний, чем потерять изменения.

Вопрос: Обязательно ли каждый раз писать git add перед git commit?
Ответ: Да. Git разделяет этапы: сначала ты выбираешь, что сохранять (add), потом сохраняешь (commit). Это позволяет не сохранять мусорные изменения.

Вопрос: Я вижу слово «репозиторий» — что это?
Ответ: Репозиторий — это просто папка, в которой ты сделал git init. Вместе со всей её историей изменений.



Самое главное, что нужно запомнить (шпаргалка)
Команда	Что делает по-человечески
git init	Начинаем следить за папкой
git status	Что изменилось?
git add файл	Этот файл хотим сохранить
git commit -m "текст"	Сохраняем новую версию
git log	Покажи историю сохранений
Правильная последовательность (выучи как таблицу умножения):

Изменяешь файлы

git add (выбираешь, что сохранять)

git commit (сохраняешь)

git log (смотришь, что получилось)


Что дальше (когда освоишь эти 5 команд)
Когда ты будешь уверенно пользоваться этими командами, добавь в копилку:

git diff — показать, что именно изменилось в файлах (строчку за строчкой)

git branch — создать параллельную версию проекта для экспериментов

git checkout — переключиться на старую версию или другую ветку

git push / git pull — синхронизироваться с GitHub (облачное хранилище)

Но сначала просто сделай 5-10 коммитов в своей тренировочной папке, чтобы команды стали привычными как дыхание.
